# Contributor Notes

## Language

Public documentation in `README.md` and `documentation/*` is written in English. Internal project
memory under `docs/*` and code comments are written in Ukrainian.

All runtime messages generated by code for users, logs, CLI output, errors, or diagnostics must be
English. This includes `EncodingError.message` and `EncodingWarning.message`.

## Examples

When the public API changes, update:

- the relevant file in `documentation/`;
- `documentation/agents.md` if the change affects agent instructions;
- `tests/public-docs-examples.test.ts` so examples compile and verify behavior.

## Public API Boundary

Public examples should import from the package root:

```ts
import { decodeDocument } from "@relicmem/encoding";
```

Do not document internal detector/decoder/source classes as an integration surface unless they are
exported as a stable public contract.

## Source-perfect behavior

Do not replace byte input with string input in parser integration examples. String input is
acceptable for tests, editor-only flows, or already-decoded workflows, but not for exact source
ranges.
